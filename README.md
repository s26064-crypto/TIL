# TIL
주석 (Comment)

주석은 프로그램 실행에는 영향을 주지 않고, 설명이나 메모를 남길 때 사용하는 것이다.

-한 줄 주석 (Single Line Comment)

// 로 시작한다.

// 뒤의 내용은 모두 주석 처리된다.

예시

// 백앤드 완전 개꿀잼


- 여러 줄 주석 (Multi Line Comment)

/* 로 시작

*/ 로 끝난다.

그 사이의 모든 내용이 주석이다.

예시

/*
백:백앤드는
앤:앤간한 버그는 다 찾아내는 직업계의
드:드라이버다
*/


(2) 자바(Java)란?

자바(Java)는 프로그래밍 언어이며
표준 스펙 + 구현으로 이루어져 있다.

(3) 자바 표준 스펙 (Standard Specification)

표준 스펙은 자바를 어떻게 만들어야 하는지에 대한 설계도이다.

특징

자바의 설계 문서

여러 회사가 이 문서를 기반으로 자바를 만든다

JCP(Java Community Process)에서 관리한다.

(4) 다양한 자바 구현

여러 회사가 자바 표준 스펙을 기반으로 실제 자바 프로그램을 만든다.

예시

Amazon Corretto

특징

회사마다 장단점이 다름

대부분 Windows / Mac / Linux 모두 지원

예

Amazon Corretto => AWS 환경에 최적화

(5) 자바 컴파일과 실행 과정

자바 프로그램은 2단계 과정을 거친다.

1 컴파일 (Compile)

개발자가 작성한

Hello.java

파일을 컴파일러가 변환한다.

.java => .class

예

javac Hello.java

결과

Hello.class

컴파일 과정에서

문법 오류 검사

JVM에서 빠르게 실행되도록 최적화

가 이루어진다.

2 실행

컴파일된 .class 파일을 실행한다.

예) java Hello

이때

JVM (Java Virtual Machine) 이 실행되면서 프로그램이 작동한다.

(6) 인텔리제이(IntelliJ)에서 컴파일과 실행

보통 직접 명령어를 사용해야 한다.

컴파일

javac Hello.java

실행

java Hello

하지만 IntelliJ는

컴파일과 실행을 자동으로 한 번에 처리해준다.

그래서 개발자가 명령어를 직접 입력할 필요가 없다.

컴파일된 파일은 보통

out 폴더

에 생성된다.

(7) 자바와 운영체제 독립성

자바의 가장 큰 특징이다.

일반 프로그램

일반 프로그램은 특정 OS에서만 실행된다.

예

Windows 프로그램 => Windows만 실행

Mac 프로그램 => Mac만 실행

이유
각 OS가 사용하는 명령어가 다르기 때문

자바 프로그램

자바는 어떤 OS에서도 실행 가능하다.

조건
=> JVM이 설치되어 있으면 된다

(8) 자바 개발 환경과 서버 환경

개발 환경과 서버 환경이 다를 수 있다.

개발: Windows, Mac

서버:Linux, AWS

예시

AWS 서버에서는

Amazon Corretto 자바를 설치해서 실행할 수 있다.

자바는 운영체제 독립성 덕분에 이런 구조가 가능하다.
