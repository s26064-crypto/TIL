# TIL
주석 (Comment)

주석은 프로그램 실행에는 영향을 주지 않고, 설명이나 메모를 남길 때 사용하는 것이다.

-한 줄 주석 (Single Line Comment)

// 로 시작한다.

// 뒤의 내용은 모두 주석 처리된다.

예시

// 백앤드 완전 개꿀잼


- 여러 줄 주석 (Multi Line Comment)

/* 로 시작

*/ 로 끝난다.

그 사이의 모든 내용이 주석이다.

예시

/*
백:백앤드는
앤:앤간한 버그는 다 찾아내는 직업계의
드:드라이버다
*/


(2) 자바(Java)란?

자바(Java)는 프로그래밍 언어이며
표준 스펙 + 구현으로 이루어져 있다.

(3) 자바 표준 스펙 (Standard Specification)

표준 스펙은 자바를 어떻게 만들어야 하는지에 대한 설계도이다.

특징

자바의 설계 문서

여러 회사가 이 문서를 기반으로 자바를 만든다

JCP(Java Community Process)에서 관리한다.

(4) 다양한 자바 구현

여러 회사가 자바 표준 스펙을 기반으로 실제 자바 프로그램을 만든다.

예시

Amazon Corretto

특징

회사마다 장단점이 다름

대부분 Windows / Mac / Linux 모두 지원

예

Amazon Corretto => AWS 환경에 최적화

(5) 자바 컴파일과 실행 과정

자바 프로그램은 2단계 과정을 거친다.

1 컴파일 (Compile)

개발자가 작성한

Hello.java

파일을 컴파일러가 변환한다.

.java => .class

예

javac Hello.java

결과

Hello.class

컴파일 과정에서

문법 오류 검사

JVM에서 빠르게 실행되도록 최적화

가 이루어진다.

2 실행

컴파일된 .class 파일을 실행한다.

예) java Hello

이때

JVM (Java Virtual Machine) 이 실행되면서 프로그램이 작동한다.

(6) 인텔리제이(IntelliJ)에서 컴파일과 실행

보통 직접 명령어를 사용해야 한다.

컴파일

javac Hello.java

실행

java Hello

하지만 IntelliJ는

컴파일과 실행을 자동으로 한 번에 처리해준다.

그래서 개발자가 명령어를 직접 입력할 필요가 없다.

컴파일된 파일은 보통

out 폴더

에 생성된다.

(7) 자바와 운영체제 독립성

자바의 가장 큰 특징이다.

일반 프로그램

일반 프로그램은 특정 OS에서만 실행된다.

예

Windows 프로그램 => Windows만 실행

Mac 프로그램 => Mac만 실행

이유
각 OS가 사용하는 명령어가 다르기 때문

자바 프로그램

자바는 어떤 OS에서도 실행 가능하다.

조건
=> JVM이 설치되어 있으면 된다

(8) 자바 개발 환경과 서버 환경

개발 환경과 서버 환경이 다를 수 있다.

개발: Windows, Mac

서버:Linux, AWS

예시

AWS 서버에서는

Amazon Corretto 자바를 설치해서 실행할 수 있다.

자바는 운영체제 독립성 덕분에 이런 구조가 가능하다.

(1) 변수(Variable)란?
=================
값을 저장하는 이름이 붙은 공간이다.

프로그램에서 데이터를 저장하고 다시 사용하기 위해 만든다.

쉽게 말하면 값을 담는 상자라고 생각하면 된다.

예시

int age = 20;

int → 정수 타입

age → 변수 이름

20 → 저장할 값

(2) 변수 선언과 초기화
-1- 변수 선언

변수를 만들기

int age;

-정수 값을 저장할 age라는 변수 생성

-2- 변수 초기화

변수에 처음 값을 넣는 것

age = 20;
-3- 선언 + 초기화 동시에
int age = 20;

실무에서 가장 많이 사용

(3) 변수 값 변경

변수는 값이 바뀔 수 있다.

int age = 20;
age = 21;

-마지막 값은 21

(4) 변수 사용
int a = 10;
int b = 20;

int sum = a + b;

결과

sum = 30

 변수에 저장된 값을 연산에 사용 가능

(5) 변수 이름 규칙
V 가능
int age
int studentAge
int student_age
X 불가능
int 1age     // 숫자로 시작
int class    // 자바 예약어
자바 권장 스타일 (카멜케이스)
studentAge
userName
orderCount
(6) 변수의 타입 (자료형)

자바는 변수를 만들 때 타입을 반드시 지정해야 한다.

예시

int age = 20;        // 정수
double pi = 3.14;    // 실수
boolean flag = true; // 참/거짓
String name = "Tom"; // 문자열

변수 타입 정리


타입	의미
int	정수	10
double	실수	3.14
boolean	참/거짓	true
String	문자열	"Hello"

(7) 변수 출력
int age = 20;
System.out.println(age);
이라는 코드를 작성하면 20이라는 값이 출력된다
